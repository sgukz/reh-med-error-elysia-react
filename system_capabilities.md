# เอกสารคุณลักษณะและความสามารถของระบบ (System Capabilities & Features)
## ระบบรายงานความคลาดเคลื่อนทางยา โรงพยาบาลร้อยเอ็ด (REH Medication Error Reporting System)

เอกสารฉบับนี้จัดทำขึ้นเพื่อรวบรวมคุณลักษณะ โครงสร้างการทำงาน และความสามารถทั้งหมดของระบบรายงานความคลาดเคลื่อนทางยา (Medication Error) โดยอ้างอิงจากความสามารถของระบบหน้าบ้าน (Frontend) และโครงสร้างการเชื่อมต่อระบบหลังบ้าน (Backend API) เพื่อใช้เป็นข้อมูลอ้างอิงสำหรับ AI หรือผู้ประเมินในการจัดทำเอกสารประเมินประสิทธิภาพระบบ (System Assessment / Evaluation Documents) หรือเอกสารการรับรอง HA (Hospital Accreditation)

---

## 1. ข้อมูลภาพรวมระบบ (System Overview)
* **ชื่อระบบ**: REH Medication Error Reporting System (ระบบรายงานความคลาดเคลื่อนทางยา รพ.ร้อยเอ็ด)
* **วัตถุประสงค์**: เพื่อใช้ในการบันทึก ติดตาม วิเคราะห์ และทำแนวทางแก้ไขปัญหาความคลาดเคลื่อนทางยาในหอผู้ป่วยและห้องยา ช่วยเพิ่มความปลอดภัยสูงสุดในการใช้ยาของผู้ป่วย
* **กลุ่มเป้าหมาย**: เภสัชกร, พยาบาล, แพทย์, เจ้าหน้าที่ห้องยา และผู้บริหารของโรงพยาบาลร้อยเอ็ด (รันบนระบบเครือข่ายภายใน Intranet)
* **สถาปัตยกรรมเทคโนโลยี**:
  * **Frontend**: React 18, Material UI (MUI v5), React Hook Form, ApexCharts, SheetJS (xlsx)
  * **Backend**: ElysiaJS (TypeScript) บน Bun Runtime ประสิทธิภาพสูง
  * **ฐานข้อมูล**: MariaDB / MySQL (เชื่อมต่อกับฐานข้อมูล HIS ของโรงพยาบาลเพื่อดึงข้อมูลแพทย์/รายการยา/ข้อมูลผู้ป่วย และเก็บข้อมูลระบบรายงานแยกอิสระ)

## 2. โครงสร้างความปลอดภัยของระบบ (Security & Access Control)
ระบบออกแบบโดยยึดหลัก **Defense-in-Depth** ตามมาตรฐานความปลอดภัย OWASP Top 10 เพื่อจำกัดสิทธิ์การเข้าถึงข้อมูลที่ละเอียดอ่อนของผู้ป่วย:
* **ระบบพิสูจน์ตัวตน (Authentication)**: 
  * เชื่อมต่อสิทธิ์การล็อกอิน (Login) ร่วมกับผู้ใช้งานของ HIS (opduser) ของโรงพยาบาล
  * จัดการเซสชันแบบปลอดภัยสูงสุดด้วย **HTTP-only Cookie (access_token)** ป้องกันการรั่วไหลผ่านการโจมตีประเภท XSS
* **การควบคุมสิทธิ์ตามบทบาท (Role-Based Access Control - RBAC)**:
  * **ผู้ใช้ทั่วไป (Staff / Nurse / Pharmacist - Rule < 9)**: 
    * สามารถบันทึกข้อมูล ดูรายการ และดูรายงานทั่วไปได้
    * มีสิทธิ์การแก้ไขหรือลบ **เฉพาะรายการที่ตนเองเป็นผู้บันทึกเท่านั้น**
  * **ผู้ดูแลระบบ (Admin / Safety Officer - Rule === 9)**:
    * มีสิทธิ์เข้าใช้งานเมนูตั้งค่าข้อมูลหลัก (Master Data) ทั้งหมด
    * สามารถแก้ไขหรือลบรายการความคลาดเคลื่อนทางยาได้ทุกรายการ
    * สามารถทำบันทึกและวิเคราะห์หาสาเหตุเชิงลึก (RCA - Root Cause Analysis) ได้
    * สามารถกรอกข้อมูลจำนวนใบสั่งยาและวันนอนของผู้ป่วย (Volume Data) ประจำปีงบประมาณ
    * สามารถปรับปรุงเกณฑ์การคำนวณโอกาสเกิดความถี่ (Likelihood Criteria) ได้
* **ความปลอดภัยในการเชื่อมต่อ (Network & Injection Guard)**:
  * ระบบตรวจสอบ `Origin` และ `Referer` กับ Allowlist ก่อนอนุญาตให้ติดต่อ API
  * ป้องกัน SQL Injection ด้วยการใช้ parameterized query ผ่าน Knex query builder
  * ป้องกันการสวมสิทธิ์การทำธุรกรรมหรือส่งค่าตัวเลขติดลบ (Sanitization) ที่ฝั่ง Client ก่อนยิง API

## 3. คุณลักษณะของแต่ละหน้าจอและการทำงาน (Screen-by-Screen Features)

### 3.1 หน้าเข้าสู่ระบบ (Login Page)
* ล็อกอินผ่านรหัสประจำตัวเภสัชกร/เจ้าหน้าที่ของโรงพยาบาลที่เชื่อมโยงกับฐานข้อมูล HIS
* มีมาตรการตรวจสอบความเข้ากันได้ของเบราว์เซอร์ พร้อมเก็บเซสชันในหน่วยความจำที่ปลอดภัยและคุกกี้ที่ไม่สามารถเข้าถึงได้ผ่านสคริปต์ (Http-Only)

### 3.2 แผงควบคุมสถิติผู้บริหาร (Executive Dashboard)
หน้าจอรายงานสรุปแบบภาพรวมสำหรับผู้บริหารแบบ Real-time แสดงผลข้อมูลด้วยกราฟที่สวยงามและแอนิเมชันที่ลื่นไหล (ApexCharts):
* **การ์ดสรุปตัวเลขสำคัญ (Summary Cards)**:
  * จำนวนอุบัติการณ์ความคลาดเคลื่อนทางยาทั้งหมด (Total Medication Errors)
  * จำนวนเคสความคลาดเคลื่อนระดับรุนแรง (Severe Errors - ระดับ E ขึ้นไป)
  * จำนวนเคสที่เกิดจากยาที่มีความเสี่ยงสูง (High Alert Drugs - HAD)
  * อัตราส่วนและเคสที่ได้รับการแก้ไขปัญหาเรียบร้อยแล้ว
* **กราฟแสดงแนวโน้มและสัดส่วน (Interactive Charts)**:
  * กราฟเส้นแสดงแนวโน้มความคลาดเคลื่อนทางยาจำแนกรายเดือน
  * กราฟแท่งแสดงสัดส่วนการเกิดเหตุการณ์แยกตามประเภทการคลาดเคลื่อน (6 ประเภท)
  * กราฟวงกลม (Pie Chart) แสดงสัดส่วนตามระดับความรุนแรง (Severity Levels A-I)
  * รายการอันดับหอผู้ป่วยหรือหน่วยงาน (Wards) ที่พบเหตุการณ์บ่อยที่สุด เพื่อชี้เป้าความเสี่ยง
  * รายชื่อยาสามัญ (Generic Name) ที่เกิดความคลาดเคลื่อนสูงสุดในกลุ่ม HAD และ Non-HAD

### 3.3 ระบบลงทะเบียนและจัดการข้อมูลความคลาดเคลื่อนทางยา (Medication Error Management)
เป็นหัวใจหลักของระบบ ประกอบด้วยระบบตารางแสดงรายการ และฟอร์มลงทะเบียนแบบไดนามิก (Dynamic Form):
* **ระบบตารางรายการ (Incident Listing Table)**:
  * แถบเลื่อนแนวนอนแบบคู่อัจฉริยะ (Dual Scrollbar) ทั้งด้านบนและล่างตาราง สะดวกแก่การเลื่อนดูข้อมูลบนหน้าจอคอมพิวเตอร์โดยไม่ต้องเลื่อนลงล่างสุด
  * คอลัมน์ตารางมีระยะกว้างขั้นต่ำ (minWidth) ป้องกันคำทับซ้อนและจัดระเบียบข้อมูลตามระดับความสำคัญ
  * แถวข้อมูลไฮไลต์สีพื้นหลังอ่อนๆ ตามระดับความรุนแรงเพื่อความชัดเจนทางสายตา:
    * ระดับ A-D: สีเขียวอ่อน (ไม่อันตราย/ไม่ถึงตัว)
    * ระดับ E-F: สีเหลืองอ่อน (เกิดอันตรายชั่วคราว/ต้องให้การรักษาเพิ่ม)
    * ระดับ G-I: สีแดงอ่อน (อันตรายถาวร/เกือบเสียชีวิต/เสียชีวิต)
  * **ระบบตัวกรองข้อมูลเชิงลึก (Advanced Filters)**: ค้นหาด้วยคำสำคัญ, ช่วงวันที่เกิดเหตุ, ค้นหาแยกตามหน่วยงาน/สถานที่พบ, ค้นหาตามประเภทการเกิดความคลาดเคลื่อน (6 ประเภทหลัก), ค้นหาตามระดับความรุนแรง (A-I), และระบุเฉพาะกลุ่มยาความเสี่ยงสูง (HAD)
* **ฟอร์มบันทึกข้อมูลอัจฉริยะ (Dynamic Medication Error Form)**:
  * แยกประเภทแบบฟอร์มบันทึกออกเป็น 2 กลุ่มชัดเจนตามพื้นที่ปฏิบัติงาน:
    1. แบบฟอร์มสำหรับโรงพยาบาลทั่วไป (General Wards)
    2. แบบฟอร์มสำหรับกลุ่มงานเภสัชกรรม (Pharmacy Department)
  * **การเชื่อมโยงระบบ HIS แบบ Real-time**: 
    * ค้นหาข้อมูลผู้ป่วยแบบทันทีโดยใช้รหัส HN หรือ AN เพื่อดึงข้อมูลเบื้องต้น
    * ค้นหาและทำ Autocomplete รายชื่อแพทย์ผู้สั่งใช้ยาจากระบบ HIS
    * ค้นหาและทำ Autocomplete รายการยาสามัญและยาทางการค้าจากระบบคลังยาของ HIS พร้อมดึงรหัสยา (icode) และหน่วยนับอัตโนมัติ
  * **ระบบกรอกข้อมูลความคลาดเคลื่อนตามขั้นตอนการใช้ยา (6 Stages of Medication Use)**:
    1. **Prescribing error (ความคลาดเคลื่อนในการสั่งใช้ยา)**: บันทึกข้อมูลการสั่งยาผิดตัว ผิดขนาด หรือสั่งผู้ป่วยผิดคน พร้อมเก็บข้อมูลชื่อแพทย์ผู้สั่งใช้
    2. **Transcribing error (ความคลาดเคลื่อนในการคัดลอกคำสั่งยา)**: เปรียบเทียบข้อมูลยาระหว่างตัวที่สั่งถูกกับตัวที่คัดลอกผิด พร้อมหน่วยนับและรหัสยา พร้อมบันทึกผู้คัดลอก
    3. **Processing error (ความคลาดเคลื่อนในการจัดเตรียม/คีย์ข้อมูล)**: บันทึกจับคู่ยาที่คีย์เข้าระบบถูกเทียบกับยาที่คีย์ผิด เพื่อวิเคราะห์คู่ยาที่มักคีย์สับสน พร้อมบันทึกผู้คีย์
    4. **Dispensing error (ความคลาดเคลื่อนในการจ่ายยา)**: บันทึกข้อมูลการจัดยาผิด จ่ายยาผิดประเภท หรือส่งมอบผิดหอผู้ป่วย พร้อมระบุชื่อเภสัชกร/เจ้าหน้าที่ผู้จัดและผู้จ่ายยา
    5. **Pre-administration error (ความคลาดเคลื่อนในการเตรียมบริการยา)**: บันทึกความผิดพลาดก่อนการบริหารยาในหอผู้ป่วย
    6. **Administration error (ความคลาดเคลื่อนในการบริหารยา/ให้ยา)**: บันทึกข้อมูลการให้ยาผิดคน ผิดเวลา หรือผิดวิธีให้กับผู้ป่วยบนหอผู้ป่วย
  * **ส่วนการบันทึกการแก้ไขปัญหาเบื้องต้นและการวิเคราะห์สาเหตุ (Initial Corrective Actions & Cause Analysis)**:
    * มี Dropdown ให้เลือกหมวดหมู่การวิเคราะห์สาเหตุหลักที่เกิดขึ้น (เช่น เกิดจากสับสนชื่อยา, อ่านลายมือแพทย์ผิด, ความล้าของเจ้าหน้าที่, ปัญหาเชิงระบบ ฯลฯ)
    * ช่องบันทึกแนวทางแก้ไขปัญหาเฉพาะหน้า
  * **โมดูลการทบทวนหาสาเหตุเชิงลึก (RCA - Root Cause Analysis Module)**:
    * ปรากฏเฉพาะระดับผู้ดูแลระบบ (Admin) สำหรับเคสความคลาดเคลื่อนที่มีระดับความรุนแรงสูง
    * ช่องวิเคราะห์หาสาเหตุที่แท้จริงและแนวทางป้องกันระยะยาว
    * บันทึกชื่อผู้ทำ RCA และบันทึกวันที่แก้ไขข้อมูลล่าสุดอัตโนมัติ

### 3.4 ศูนย์รายงานอุบัติการณ์และความคลาดเคลื่อน (Reports Center)
ประกอบด้วยหน้ารายงานย่อยทั้งหมด 10 รูปแบบ และรายงานคู่ยาคลาดเคลื่อน รองรับการส่งออกข้อมูลเป็นไฟล์ Excel (.xlsx) ที่มีการแยกสเปรดชีตอย่างสวยงาม:
1. **รายงานสรุปข้อมูลแยกตามหน่วยงาน (Report Summary 1)**: ดูยอดรวมอุบัติการณ์ของแต่ละหอผู้ป่วย/ห้องยา
2. **รายงานสรุปจำแนกตามประเภทความคลาดเคลื่อนและหอผู้ป่วย (Report Summary 2)**: แผนภูมิการกระจายตัวของประเภทข้อผิดพลาดในแต่ละหอผู้ป่วย
3. **รายงานตามผู้สั่งใช้ยา/ผู้เตรียมยา/ผู้บริหารยา (Report Summary 3)**: วิเคราะห์รายบุคคลและบทบาทหน้าที่ของผู้เกี่ยวข้อง
4. **รายงานแยกตามกลุ่มพนักงาน (Report Summary 4)**: สรุปยอดสะสมข้อผิดพลาดจำแนกตามวิชาชีพ (แพทย์, เภสัชกร, พยาบาล, เจ้าพนักงานเภสัชกรรม ฯลฯ)
5. **รายงานจำแนกตามระดับความรุนแรงและหอผู้ป่วย (Report Summary 5)**: สรุปความเสี่ยงระดับรุนแรง (Severe Case Matrix) แยกรายหอผู้ป่วย
6. **รายงานสรุปอุบัติการณ์ที่ได้ RCA แล้ว (Report Summary 6 - RCA Report)**:
   * **จุดเด่น**: รายงานเชิงคุณภาพสูง แสดงข้อมูล 17 คอลัมน์ที่ครบถ้วน (ประวัติเหตุการณ์, ผลการแก้ไข, ตัวยา, ชื่อแพทย์, ผู้ทบทวน ฯลฯ)
   * **KPIs Cards**: บอร์ดสรุป 6 ค่าสำคัญ (จำนวน RCA ทั้งหมด, อัตราส่วนความรุนแรง E+, กลุ่มยา HAD, ระยะเวลานำเฉลี่ยในการปิดเคส RCA (วัน), ประเภทความเสี่ยงอันดับ 1, และหอผู้ป่วยที่พบเหตุสูงสุด)
   * **RCA Response Time Color-coded Chip**: แสดงระยะเวลาที่ใช้ในการทบทวน RCA (สีเขียว <= 7 วัน, สีเหลือง <= 30 วัน, สีแดง > 30 วัน) เพื่อประเมินความเร็วในการรับมือกับความเสี่ยง
   * **Excel Export**: ส่งออก 2 แผ่นงาน (Sheet 1: ตารางสรุปสถิติ, Sheet 2: รายการละเอียดเคสทั้งหมด)
7. **รายงานสรุปจำนวนการเกิดแยกตามเวรปฏิบัติงาน (Report Summary 7)**: สรุปอุบัติการณ์แยกตาม เวรเช้า, เวรบ่าย, และเวรดึก เพื่อวิเคราะห์หาช่วงเวลาวิกฤต
8. **รายงานสรุปรายการยาที่พบบ่อย (Report Summary 8)**: จัดลำดับชื่อยาที่ถูกสั่งหรือจัดจ่ายผิดบ่อยที่สุด แยกกลุ่ม HAD ชัดเจนเพื่อทำมาตรการป้องกันความสับสน (LASA - Look-Alike Sound-Alike)
9. **รายงานรายละเอียดความคลาดเคลื่อนแยกประเภทและประเมินระดับความถี่โอกาสเกิด (Report Summary 9 - Likelihood Auto-Report)**:
   * **จุดเด่น**: ระบบเปรียบเทียบข้อมูล 2 ช่วงเวลา (ช่วงเวลา A เทียบกับ ช่วงเวลา B พร้อมคำนวณผลต่างเป็นเปอร์เซ็นต์ Δ% เพิ่ม/ลด)
   * **คะแนน Likelihood แบบคำนวณอัตโนมัติ (Auto-Likelihood)**: ระบบหลังบ้านจะคำนวณคะแนนโอกาสเกิด (Score 1-5) ให้อัตโนมัติจากความถี่ของแต่ละ Subtype ยาในช่วงเวลานั้นๆ โดยเปรียบเทียบกับตารางเกณฑ์ Likelihood และนำเสนอคะแนนเป็น Chip สีตามระดับ:
     * 5 = สีแดง (Frequent / เกิดบ่อยมาก)
     * 4 = สีส้ม (Likely / เกิดบ่อย)
     * 3 = สีเหลืองอำพัน (Possible / อาจเกิด)
     * 2 = สีเขียวอ่อน (Unlikely / ไม่ค่อยเกิด)
     * 1 = สีเขียว (Rare / เกิดน้อยมาก)
     * 0 = สีเทา (Never / ไม่เคยเกิดเลย)
10. **รายงานสถิติจำนวนใบสั่งยา/วันนอน (Report Summary 10)**:
    * **จุดเด่น**: ใช้คำนวณหาอัตราการเกิดอุบัติการณ์ต่อ 1,000 ใบสั่งยา หรือต่อ 1,000 วันนอน (Incident Rate per 1,000 Units) เพื่อเปรียบเทียบมาตรฐานความปลอดภัยตามเกณฑ์สากล
    * **Table 0 (Volume Data)**: หน้าจอให้แอดมินกรอกตัวเลขจำนวนวันนอนผู้ป่วยใน (IPD Bed-days) และจำนวนใบสั่งยาผู้ป่วยนอก (OPD Prescriptions) แยกตามแต่ละเดือนของปีงบประมาณไทย (ตุลาคม - กันยายน)
    * **Table A & B (IPD/OPD Error Counts)**: ตารางจำแนกการเกิดความคลาดเคลื่อน 6 ประเภท รายเดือนแยกกลุ่ม HAD และ Non-HAD
    * **Table C (Rate calculation)**: แสดงผลการคำนวณอัตราความคลาดเคลื่อนต่อ 1,000 หน่วยในแต่ละเดือนแบบอัตโนมัติ
11. **รายงานคู่ยาคลาดเคลื่อน (Drug Pair Summary Report)**: รายงานวิเคราะห์คู่ยาที่มีการจ่ายสลับหรือคีย์สับสนกันสูงสุด (เช่น ยา A สับสนกับ ยา B) ช่วยจัดกลุ่มยาสุ่มเสี่ยงเพื่อแยกจุดจัดเก็บในคลังยา

### 3.5 หน้าจอกำหนดเกณฑ์โอกาสเกิดความคลาดเคลื่อน (Likelihood Criteria Page)
เครื่องมือระดับสูงสำหรับแอดมินในการควบคุมเกณฑ์และคะแนนประเมินความเสี่ยง:
* **ระบบ 6 แท็บตามประเภทข้อผิดพลาด (6 tabs by Error Types)**: แอดมินสามารถกำหนดเกณฑ์จำนวนครั้งการเกิดเหตุ (min-max) สำหรับให้คะแนน Likelihood (1-5) แยกเป็นอิสระต่อกันทั้ง 6 ประเภท (สั่งยา, คัดลอก, จัดเตรียม, จ่ายยา, เตรียมบริหาร, บริหารยา)
* **Visual Range Bar**: แถบแสดงแถบสีเปรียบเทียบช่วงตัวเลข (Range Bar) ของแต่ละระดับคะแนนแบบเรียลไทม์ ช่วยให้ผู้ใช้งานเห็นภาพรวมของขนาดเกณฑ์
* **ระบบคัดกรองข้อผิดพลาดข้อมูลเรียลไทม์ (Validation & Integrity Engine)**:
  * ตรวจสอบช่องว่างข้อมูล (Gap) ที่ไม่มีระดับคะแนนใดรองรับ
  * ตรวจสอบช่วงที่ทับซ้อนกัน (Overlap) ของคะแนนแต่ละระดับ
  * ตรวจสอบข้อมูลขาดหาย (Missing) หรือการกำหนดค่า Min เกินกว่า Max
  * แสดงกล่องเตือนสีแดง (Alert Banner) ชี้จุดที่มีข้อผิดพลาดพร้อมบล็อกปุ่มบันทึก เพื่อป้องกันความเสียหายของสูตรคำนวณในระบบ

### 3.6 ระบบจัดการข้อมูลหลัก (Master Data Management Pages)
กลุ่มหน้าจอสำหรับแอดมินในการดูแลรักษาฐานข้อมูลของระบบให้เป็นปัจจุบันอยู่เสมอ มาพร้อมสถิติจำนวนรายการและระบบล้างตัวกรองด่วน:
* **จัดการหอผู้ป่วย/หน่วยงาน (DepartmentPage)**: 
  * เพิ่ม ลบ แก้ไขหน่วยงาน พร้อมระบุรหัสหน่วยงาน (`depcode`) ของ HIS
  * ตัวกรองด่วนเพื่อคัดแยกกลุ่มหอผู้ป่วย เช่น OPD, IPD, งานเตรียมยาเคมีบำบัด, งานเตรียมสารอาหารทางหลอดเลือด (TPN) ฯลฯ
  * แสดงสถานะการเปิด/ปิดใช้งาน (Active / Inactive) พร้อมตัวเลขจำนวนรายการ
* **จัดการผู้รับผิดชอบ/บุคลากร (OfficerPage)**: 
  * เพิ่ม ลบ แก้ไขรายชื่อเจ้าหน้าที่ผู้เกี่ยวข้องในการบริหารยาและจ่ายยา
  * ตัวกรองด่วนคัดแยกตามตำแหน่งวิชาชีพ (เภสัชกร, เจ้าพนักงานเภสัชกรรม, พนักงานประจำห้องยา ฯลฯ) พร้อมนับจำนวนแยกตำแหน่ง
* **จัดการประเภทความคลาดเคลื่อน (ErrorTypePage)**: 
  * จัดการประเภทย่อยของความคลาดเคลื่อนในแต่ละขั้นตอนหลัก (6 ขั้นตอน)
  * กำหนดค่า **คะแนนระดับความรุนแรง/ผลกระทบ (Impact Score: 1-5)** ของแต่ละประเภทย่อย เพื่อนำไปไขว้กับคะแนน Likelihood ในการวิเคราะห์ระดับความเสี่ยงตามมาตรฐานสากล
  * มีระบบตรวจจับรายการที่ยังขาดการกำหนดคะแนน Impact เพื่อเตือนให้กรอกข้อมูลให้ครบถ้วน
* **จัดการการวิเคราะห์สาเหตุ (AnalysisPage)**: 
  * บันทึกหัวข้อการวิเคราะห์สาเหตุ (Root Causes) เพื่อให้เจ้าหน้าที่เลือกใช้ในฟอร์มรายงาน

## 4. ตารางสรุปฟีเจอร์เด่นเพื่อการประเมิน (Key Feature Highlights for Evaluation)

| ฟีเจอร์หลัก (Core Feature) | รายละเอียดทางเทคนิคและ UX/UI (Technical & UX/UI Details) | ประโยชน์ทางคลินิกและการบริหาร (Clinical & Admin Benefits) |
|---|---|---|
| **HIS Integration** | เชื่อมข้อมูลผู้ป่วย แพทย์ และรายการยาจากระบบฐานข้อมูลโรงพยาบาลโดยตรงผ่าน Autocomplete | ลดระยะเวลาการกรอกฟอร์ม ป้องกันข้อมูลสะกดผิด และลดภาระงานของเจ้าหน้าที่ |
| **Dynamic Form Division** | ฟอร์มปรับตัวและกรองข้อมูลตามฝ่ายงาน (โรงพยาบาลทั่วไป vs ฝ่ายเภสัชกรรม) | ทำให้หน้าฟอร์มกระชับ แสดงผลเฉพาะส่วนที่เกี่ยวข้องกับผู้ใช้งานกลุ่มนั้นๆ |
| **6-Stage Error Logging** | เก็บบันทึกข้อมูลละเอียดแยกตาม 6 ขั้นตอนกระบวนการใช้ยา (Prescribing to Administration) | ช่วยชี้เป้าขั้นตอนที่เป็นจุดอ่อนที่สุดในห่วงโซ่การบริหารยาเพื่อออกนโยบายแก้ไข |
| **Auto-Likelihood Engine** | คำนวณความถี่และแปลงเป็นคะแนน Likelihood (1-5) อัตโนมัติด้วยระบบหลังบ้าน | ประเมินระดับความเสี่ยงได้อย่างเที่ยงตรง แม่นยำตามเกณฑ์สถิติจริง ไม่ขึ้นกับดุลยพินิจส่วนตัว |
| **Visual Validation Engine** | ระบบวิเคราะห์ช่องว่างและช่วงซ้อนทับ (Gap/Overlap) ของข้อมูลเกณฑ์ประเมินแบบเรียลไทม์ | ป้องกันความผิดพลาดในการป้อนข้อมูลเกณฑ์ความปลอดภัยของแอดมิน เพื่อความเสถียรของสูตรคำนวณ |
| **RCA Lead-time Monitoring** | สรุปผลความเร็วในการวิเคราะห์และทบทวน RCA (ระดับรุนแรง E ขึ้นไป) พร้อมแถบสีแจ้งเตือน | กระตุ้นให้เกิดกระบวนการเรียนรู้และแก้ไขปัญหาอย่างรวดเร็ว (ความเสี่ยงสูงต้องแก้ไขในเวลาที่กำหนด) |
| **Incident Rate per 1,000** | ประมวลผลอุบัติการณ์เทียบกับจำนวนใบสั่งยาและวันนอนของผู้ป่วยแบบจำแนกรายเดือน | ชี้วัดระดับคุณภาพความปลอดภัยของหอผู้ป่วยตามมาตรฐานสากล (ใช้ประกอบการประเมินคุณภาพโรงพยาบาล HA) |
| **Dual Sheet Excel Export** | ส่งออกข้อมูลดิบและตารางสรุปทางสถิติแยกเป็นคนละแผ่นงานในไฟล์ Excel ไฟล์เดียว | สะดวกต่อการนำข้อมูลไปวิเคราะห์ต่อหรือทำรายงานเสนอผู้บริหารระดับสูงโดยไม่ต้องจัดหน้าใหม่ |
| **Premium Responsive UI** | หน้าจอสไตล์ Glassmorphism คอนโทรลธีมสีสบายตา รองรับการใช้งานมือถือ แท็บเล็ต และคอมพิวเตอร์ | เพิ่มอัตราการยอมรับและการใช้งานระบบของเจ้าหน้าที่ (User Adoption Rate) |
